# SprintLens UI 레이아웃 가이드

이 문서는 SprintLens 웹 UI의 레이아웃 구조와 디자인 규칙을 정의한다.
모든 페이지는 이 가이드를 따라 일관된 UI를 유지해야 한다.

## 전체 레이아웃 구조

```
+------------------------------------------------------------------+
| Navbar (상단 고정)                                                 |
| [SprintLens 로고]  [메뉴1] [메뉴2] [...]           [우측 액션 영역] |
+------------------------------------------------------------------+
| Sidebar (좌측 고정)  |  Main Content (스크롤 가능)                  |
| 240px 너비           |  나머지 전체                                 |
|                      |                                              |
| - 컨텍스트 네비게이션  |  페이지별 콘텐츠 영역                        |
| - 필터, 목록 등       |  (#main-content)                             |
|                      |                                              |
+----------------------+----------------------------------------------+
```

### 영역별 역할

| 영역 | 역할 | 높이/너비 | 스크롤 |
|------|------|-----------|--------|
| **Navbar** | 로고, 탑 메뉴, 우측 액션 | 높이 64px (`h-16`) | 없음 (고정) |
| **Sidebar** | 페이지별 보조 네비게이션 | 너비 240px (`w-60`) | 독립 스크롤 |
| **Main Content** | 핵심 콘텐츠 영역 | 나머지 전체 | 독립 스크롤 |

## Navbar 상세

```
[SprintLens]  [대시보드] [프로그램팀 일정] [...]              [새로고침]
 ↑ 로고/홈      ↑ 탑 메뉴 (좌측 정렬)                    ↑ 우측 액션
```

- **로고 (SprintLens)**: 클릭 시 홈 화면으로 이동
- **탑 메뉴**: 주요 페이지 전환. 활성 메뉴에 하단 보더 또는 배경 강조
- **우측 액션**: 새로고침 등 글로벌 액션 버튼
- 스타일: `bg-base-100 border-b border-base-300`
- 메뉴 간격: `gap-1`, 버튼 스타일: `btn btn-ghost btn-sm`

## Sidebar 상세

- 너비: `w-60` (240px)
- 배경: `bg-base-100`
- 우측 보더: `border-r border-base-300`
- 높이: Navbar 아래 나머지 전체 (`h-[calc(100vh-64px)]`)
- 오버플로: `overflow-y-auto`
- 내부 패딩: `p-4`
- 페이지별 다른 콘텐츠를 표시 (필터, 서브 메뉴, 목록 등)
- 콘텐츠가 없는 페이지에서도 사이드바 영역은 유지

## Main Content 상세

- Sidebar 오른쪽의 나머지 영역 전체 사용
- 높이: Navbar 아래 나머지 전체 (`h-[calc(100vh-64px)]`)
- 오버플로: `overflow-y-auto`
- 내부 패딩: `p-6`
- 최대 너비 제한 없음 (사이드바가 이미 너비를 차지)
- HTMX로 `#main-content` 영역만 교체

## 페이지 구성

### 홈 (/)
- 초기 화면. 빠르게 로드되는 가벼운 페이지
- Jira API 호출 없음
- 주요 메뉴 바로가기 카드 등 표시
- 사이드바: 빈 상태 또는 최근 활동 요약

### 대시보드 (/dashboard)
- 스프린트 현황 전체 데이터 (Jira API 호출)
- Stats 카드 + 진행률 + 담당자별/스토리별 현황 + 일감 리스트
- 사이드바: 스프린트 선택, 필터 등 (추후 확장)

### 프로그램팀 일정 (/schedule)
- 프로그램팀 일정 관련 콘텐츠
- 사이드바: 월/주 선택 등 (추후 확장)

## HTMX 패턴

- 탑 메뉴 클릭 → `hx-get="/partials/{page}"` → `#main-content` 교체
- 사이드바 콘텐츠도 페이지별 파셜로 교체 가능 (`#sidebar-content`)
- 자동 갱신: 필요한 페이지에서만 `hx-trigger="every 300s"` 적용

## 색상 및 스타일 규칙

- DaisyUI `light` 테마 사용
- 카드: `bg-base-100 rounded-2xl shadow-sm border border-base-300`
- 배지 색상: 완료=`badge-success`, 진행중=`badge-info`, 대기=`badge-ghost`
- 텍스트 보조색: `text-base-content/60`
- 반응형: 모바일에서 사이드바 숨김 (`hidden lg:block`)
